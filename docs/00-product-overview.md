# 제품 개요 (Product Overview)

## 한 줄
AI랑 카카오톡처럼 대화하는 도트 캐릭터 컴패니언 앱 (iOS + Android).

## 차별점 — "AI 웹챗"이 아니라 "사람과 톡하는 느낌"

| AI 웹챗 (ChatGPT/Claude) | agenttalk (목표) |
|---|---|
| 한 번에 긴 블록 하나 | 짧은 메시지 여러 개가 다다닥 |
| 마크다운·불릿·헤더 | 줄임말·이모티콘·스티커 |
| 즉답, 시간감 없음 | 입력 중… 뜸 들이다 톡 |
| "도와드릴게요" 비서체 | "헐 진짜?ㅋㅋ" 친구체 |
| 내가 말해야 답함 | 먼저 연락 옴 |
| 답을 "완성"하려 함 | 대화는 안 끝남 |

이 **"메시징 느낌"이 차대(chassis)**고, 페르소나·모드(힐링/수다/응원)는 그 위에 올라탄다. 차대가 진짜면 뭘 올려도 살아난다.

## 기획 결정 사항
- **플랫폼**: iOS + Android 크로스플랫폼 (Flutter)
- **자유도 = 3축**: ① UI·테마 꾸미기 ② 멀티 캐릭터·단톡방 ③ 대화 제약 최소화
  - 사용자 캐릭터 직접 제작·공유(UGC 마켓플레이스)는 범위 외 → 초기 범위 축소
- **귀여움**: 도트(픽셀아트) + 통통 튀는 UX

## 기술 스택
- 앱: **Flutter** / 백엔드: **Supabase** + LLM 오케스트레이션 함수 / LLM: **프론티어**(프로토타입)
- 캐릭터 에셋: **sprite-gen** (디자인타임, 출력 = 스프라이트 시트 PNG + `manifest.json`)
  - ⚠️ sprite-gen은 앱 런타임이 아니라 **오프라인 에셋 파이프라인**. 산출물을 앱에 커밋해두고 Flutter가 렌더.
- ⚠️ 앱에서 LLM 직접 호출 금지 — 키 노출·비용·정책·기억 주입 때문에 백엔드 함수 경유.

## 설계 철학
"반복 가능한 건 AI가, 판단은 사람이." — sprite-gen이 프레임을 생성하고, 큐레이션(회전·스케일·위치)은 사람이 한다.

## 리스크 (조기 결정 필요)
- **대화 제약 최소화 ↔ 앱스토어 정책 ↔ 한국 법**: iOS는 성인/수위 콘텐츠에 민감(연령등급·심사 거부). "허용/금지 라인"을 LLM 제공자 선택 *전에* 한 줄로 정의.
- **비용**: 단톡방 = 메시지 1개에 캐릭터 N명 응답 → 토큰 N배. 무료/유료 경계를 처음부터 설계.

## 빌드 순서 (MVP)
1. 캐릭터 1명 페르소나 확정
2. 베이스 도트 1장 → 액션(감정) JSON → sprite-gen → 스프라이트 아틀라스
3. Flutter 수직 슬라이스: 채팅 1화면 + 버블 엔진 + 백엔드 1엔드포인트 + 프론티어 LLM
4. "느낌" 검증 → 귀여움 레이어(모션·햅틱·사운드) → 상태 모델(무드/호감도 시각화)
